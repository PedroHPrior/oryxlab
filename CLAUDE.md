# CLAUDE.md

Project-specific notes for Claude Code (and any other AI coding agent that
honors this convention) working in the OryxLab repository.

## What this project is

**OryxLab** is a DPS calculator and build optimizer for *Realm of the Mad
God*. It runs entirely in the browser, with a small Express backend that
serves the bundled SPA and proxies the RealmEye player API. The DPS engine
is pure-functional and lives in `src/engine/`.

It is **not** a generic React template, a Design OS planning tool, or a
shadcn starter — even though traces of those origins exist in the git
history. Treat the current `src/` as the source of truth.

## Architecture at a glance

```
browser (Vite SPA)
  └─ src/app/         OryxLabApp + routes + shared state (useOryxLab)
  └─ src/engine/      pure-functional DPS + beam-search optimizer (Worker)
  └─ src/sections/    feature views (comparator, catalog, optimizer, …)
  └─ src/shell/       AppShell + nav chrome
  └─ src/lib/         router + Suspense wrappers

server/index.mjs      Express 5 (helmet + rate-limit + gzip)
  ├─ /api/items       static catalog JSON
  ├─ /api/classes     static class JSON
  ├─ /api/realmeye/*  authenticated 5-minute-cached proxy
  └─ *                static SPA fallback

product/data/         source-of-truth catalog data (items, classes, sets,
                      balance) — authored / scraped, NOT generated by build
product/sections/     per-feature TypeScript types + sample fallback data
```

## Conventions

- **Tailwind v4**, no `tailwind.config.js`. Use built-in utility classes and
  built-in colors. The brand accent is `amber-400`.
- **Strict TypeScript**, no `any` without a justification comment.
- **React Compiler** rules enforced by ESLint — manual `useMemo` /
  `useCallback` should not exist unless the React Compiler can preserve
  them. If a memoization can't be preserved, refactor (refs, reducers, …)
  rather than disabling the rule.
- **No new dependencies** without a clear functional justification — bundle
  size and supply-chain surface both matter.
- **Tests live in `tests/`** and run under Vitest (`npm test`). The DPS
  engine has tight unit coverage; UI tests are smoke-level.

## Running things

```bash
npm ci
npm run dev:all        # Vite + Express in parallel
npm test               # Vitest (165 tests)
npm run typecheck      # tsc --noEmit
npm run lint           # eslint .
npm run build          # tsc -b + vite build
npm run build:og       # regenerate public/og-image.png
```

## Editing checklist

Before committing:

1. `npm run lint` — must be 0 errors / 0 warnings.
2. `npm run typecheck` — must be clean.
3. `npm test` — must be green.
4. Don't commit `.notes/`, `.audit/`, `coverage/`, or `dist/`.
5. Don't accept external pull requests — see `CONTRIBUTING.md` and
   `.github/workflows/close-external-prs.yml`.
